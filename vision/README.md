# Vision: Executive Materials for Partners & Stakeholders

## Purpose

This directory contains **high-level, external-facing documents** designed for strategists, partners, funders, and stakeholders who need to understand the Unified Conscious Evolution Framework quickly and clearly.

**Target Audiences:**
- Impact investors and philanthropic organizations
- Institutional partners (universities, research institutes, NGOs)
- Government agencies and policy makers
- Strategic advisors and board members
- Media and journalists

## Usage Guidelines

### For UCF Contributors

**When to create external-facing documents:**
- Approaching potential partners
- Presenting at conferences
- Applying for grants or funding
- Engaging with media

**Quality standards:**
- Professionally formatted (consider PDF export)
- Free of jargon (or jargon explained)
- Evidence-based claims (cite pilots, research)
- Clear calls to action
- Visually compelling (diagrams, infographics welcome)

**Process:**
1. Draft document in this directory
2. Review with UCF core team
3. Iterate based on feedback
4. Mark as "Approved for external use" when ready

### For External Audiences

**Start here:**
1. Read [Executive Summary](executive-summary.md) first
2. If interested, review [Strategic Overview](strategic-overview.md)
3. For specific topics, see specialized documents above

**For more depth:**
- Full framework: [`docs/conscious-evolution-framework.md`](../docs/conscious-evolution-framework.md)
- Technical architecture: [`docs/metallicus-interoperability-thesis.md`](../docs/metallicus-interoperability-thesis.md)
- Repository relationships: [`docs/repository-relationships.md`](../docs/repository-relationships.md)

**For implementation details:**
- See individual pillar directories (e.g., [`pillar-1-science/`](../pillar-1-science/))
- Visit implementation repositories (e.g., [EHDC Repository](https://github.com/dj-ccs/EHDC))

## Differentiation: Vision vs. Implementation

**UCF main docs (docs/):** Internal architectural documents, detailed specifications, governance

**Implementation repos (e.g., EHDC):** Working code, technical documentation, pilot management
